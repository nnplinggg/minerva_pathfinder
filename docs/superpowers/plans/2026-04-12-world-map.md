# World Map — M4 Implementation Plan

> **For agentic workers:** Use `superpowers:subagent-driven-development` or `superpowers:executing-plans` to implement task-by-task. Each task is independently testable.

**Goal:** Replace the current `.school-grid` button list with a pixel-art top-down map in the style of Sprout Lands — grass terrain, trees, sandy paths, 5 school nodes placed geographically, 3 treasure chests that unlock progressively.

**Tech stack:** Vanilla HTML/CSS/JS. No framework, no build step. Sprout Lands UI Pack sprites as `<img>` or CSS `background-image`. `image-rendering: pixelated` throughout.

**Files changed:**
- `index.html` — map screen restructure + modal + chest HTML
- `css/styles.css` — pixel map styles, node styles, chest styles, modal, animations
- `js/app.js` — state extension, renderMap() rewrite, chest unlock logic, modal handler

**Files added (assets — user provides from Sprout Lands pack):**
- `assets/map-bg.png` — tiled grass background (or use CSS fallback)
- `assets/sprites/school-node-locked.png`
- `assets/sprites/school-node-unlocked.png`
- `assets/sprites/chest-closed.png`
- `assets/sprites/chest-open.png`
- `assets/sprites/tree.png` *(optional decorative)*

> **Asset fallback strategy:** Every task uses a CSS-only pixel placeholder first. When Sprout Lands assets are dropped into `assets/sprites/`, replacing a placeholder is a one-line `background-image` change. This means the map is fully functional before any real sprites are added.

function loadOpenedChests() {

function saveOpenedChests(opened) {

### 5d — Rewrite renderMap()

Replace existing `renderMap()` with:

```js
function renderMap() {
  showScreen('map');
  const count = state.discovered.length;

  // Update HUD
  document.getElementById('hud-count').textContent = `${count} / 5`;

  // School nodes
  Object.keys(SCHOOLS).forEach(id => {
    const node = document.querySelector(`.map-node[data-school="${id}"]`);
    const stamp = document.getElementById(`stamp-${id}`);
    if (state.discovered.includes(id)) {
      node.classList.add('discovered');
      stamp.textContent = '✅';
    } else {
      node.classList.remove('discovered');
      stamp.textContent = '';
    }
    node.onclick = () => startAdventure(id);
  });

  // Treasure chests
  document.querySelectorAll('.map-chest').forEach(el => {
    const chestNum = parseInt(el.dataset.chest, 10);
    const chest = CHESTS[chestNum - 1];
    const isOpen    = state.openedChests.includes(chest.id);
    const canUnlock = count >= chest.unlocksAt;

    el.classList.toggle('unlocked', canUnlock && !isOpen);
    el.classList.toggle('opened',   isOpen);
    el.classList.remove('just-unlocked');

    el.onclick = canUnlock && !isOpen
      ? () => openChest(chest, el)
      : null;
  });
}
```

**Test:** Map renders with all nodes and chests reflecting localStorage state. Completing an adventure and returning to map shows the node as discovered. Chest cursor changes from `not-allowed` to `pointer` at correct thresholds.

## Task 6 — Treasure chest modal (JS + CSS)

**Files:** `js/app.js`, `css/styles.css`

### JS

```js
function openChest(chest, chestEl) {
  // Mark as opened
  if (!state.openedChests.includes(chest.id)) {
    state.openedChests.push(chest.id);
    saveOpenedChests(state.openedChests);
  }

  // Update chest element
  chestEl.classList.remove('unlocked');
  chestEl.classList.add('opened');

  // Populate and show modal
  document.getElementById('modal-fact').textContent  = chest.fact;
  const link = document.getElementById('modal-link');
  link.textContent = chest.linkLabel;
  link.href        = chest.url;

  const modal = document.getElementById('chest-modal');
  modal.hidden = false;
  modal.querySelector('.chest-modal__box').focus();
}

// Wire close button (add to init or at bottom of file)
document.getElementById('modal-close').onclick = () => {
  document.getElementById('chest-modal').hidden = true;
};
```
